3D Maze Generation

Overview

For my Dissertation as part of my degree, I created algorithms to both create and solve 3D cube (n-by-n-by-n) Mazes.

As part of this project, I researched the history of mazes and maze generation, as well as real-world applications of generation and solving algorithms.

My Artefact

To create these algorithms, I applied graph theory concepts to mazes. Nodes represent junctions, while edges represent paths. A root node serves as the maze entrance, and a selected leaf node as the exit. Since trees have only one path between nodes, they naturally form "perfect" mazes (mazes without loops).

In a 3D maze, nodes can have up to five children instead of two (plus one parent). I used Primâ€™s and Depth-First Search algorithms for maze generation, and Breadth-First Search for solving.

To analyze performance, I visualized results in Tableau, comparing factors such as the number of dead ends (difficulty) and generation time (speed).

For future work, I want to create interactive 3D mazes and AI that mimics human problem-solving to better evaluate maze difficulty.
